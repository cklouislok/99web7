Our team used 3 Web Mapping Applications (Dashboard, Experience Builder and SDK)to solve the problem. Initially, we tested each application's function separately and tried to use only one application to make the solution. We found that all applications have their strengths and weaknesses. Therefore, we combine the use of three applications to create the Toronto Neighbourhood Explorer finally. Each application will be discussed on separate pages. Below is a summary table of comparison among 3 applications.

Application

Strengths

Weaknesses

Potential Uses

ArcGIS Dashboards

• Dozens of templates
• Does not require knowledge of coding
• Wide range of configurable elements for creating interactive web applications
• Integration of 2D and 3D
• Seamless integration with AGOL
• User may think its interface should be most familiar because we used to see this interface during the pandemic
• Quick response time. Table, graph, and figure update with the map extent simultaneously. 		• Lack of spatial analysis elements
• Lack of functions to locate features around a location
• Not showing source code leading to difficulties in debugging
• Cannot collaborate with others
• If you want to create 2 pages, you need to set up actions and interactions between pages. 		• In-time report system
• Weather/Transportation/Public Health information platform
• Business presentation

ArcGIS Experience Builder

• Dozens of templates
• Wide variety of configurable widgets for creating interactive and responsive web applications, inclusive of mapcentric, datacentric, page layout and design widgets
• Integration of 2D and 3D visualization
• Efficiency in configuring ready-to-use web applications
• Have spatial analysis widgets
• Does not require programming knowledge
• Seamless integration with AGOL
• Useful embed widget to embed URL/codes 		• May have performance issues for large datasets
• Widgets related to spatial analysis and infographics consume credits, e.g. analysis widget, business analyst widget
• Not showing source code leading to difficulties in debugging
• Cannot collaborate with others
• Insufficient documentation for widgets. Most documentation is text-based.
• Domain cannot be changed.
